`qubickit` (npm: `@qubics/kit`) is a TypeScript SDK for Qubic:

- Typed clients for HTTP partner API, Query V2, and JSON-RPC
- Generated contract bindings with registry-backed contract indices
- Crypto helpers for keys, identities, and transaction signing
- Works in Node 18+ and browsers via `fetchTransport`

## Build with the SDK

<Steps>
  <Step>
    <h3>Install the toolkit</h3>
    <p>
      Run <code>npm install @qubics/kit</code>, add it to your preferred package manager, and keep your TypeScript and bundler outputs aligned.
    </p>
  </Step>
  <Step>
    <h3>Configure transports once</h3>
    <p>
      Create a shared <code>fetchTransport</code> (or your own implementation), point it at the right base URLs, and reuse it across HTTP, Query, RPC, and contract bindings.
    </p>
  </Step>
  <Step>
    <h3>Ship production flows</h3>
    <p>
      Combine typed client calls, generated procedures, and crypto helpers to sign and broadcast transactions from one toolkit.
    </p>
  </Step>
</Steps>

## Quickstart

```ts
import {
  createHttpClient,
  createQueryClient,
  createRpcClient,
  fetchTransport,
  ContractTypes,
} from "qubickit";

const transport = fetchTransport();

const http = createHttpClient({ baseUrl: "https://api.qubic.org", transport });
const balance = await http.getBalance("IDENTITY");

const rpc = createRpcClient({ baseUrl: "https://rpc.qubic.org", transport });
const info = await rpc.call("getNodeInfo", []);

const query = createQueryClient({ baseUrl: "https://query.qubic.org", transport });
const txs = await query.getTransactionsForIdentity({ identity: "IDENTITY" });

// Contract function (uses registry contractIndex)
const qx = ContractTypes.Qx.createClient(query);
const orders = await qx.functions.EntityAskOrders({
  entity: new Uint8Array(32),
  offset: 0n,
  limit: 20n,
});
```

## Install

```bash
npm install @qubics/kit
# or pnpm add / bun add @qubics/kit
```

Peer dep: TypeScript >= 5. Outputs: ESM + CJS + dts.

## Clients at a glance

- **HTTP partner API**: balances, assets, broadcast, `querySmartContract`.
- **JSON-RPC**: single call + batch; errors surfaced as `QubicError`.
- **Query V2**: tick data and transactions; base64 handled for you.
- **Transports**: `fetchTransport()` by default; pass a custom fetch/transport if needed (e.g., tests or restricted envs).

## Contracts

- Generated bindings live under `ContractTypes.*` (registry-backed `contractIndex` with override).
- `functions` call `querySmartContract` and decode outputs.
- `procedures` build `{ inputType, payload }` for transaction signing/broadcast.

Example:
```ts
const qswap = ContractTypes.Qswap.createClient(query);
const pool = await qswap.functions.GetPoolBasicState({
  assetIssuer: new Uint8Array(32),
  assetName: 123n,
});

const { inputType, payload } = qswap.procedures.AddLiquidity({
  _contractIndex: qswap.contractIndex,
  _type: 0,
  assetIssuer: new Uint8Array(32),
  assetName: 123n,
  userIncreaseLiquidity: 1_000_000n,
  quAmount: 500_000n,
  assetAmount: 50_000n,
  _terminator: 0,
});
// Sign and broadcast using transaction helpers.
```

## Crypto & identities

- Keys: `deriveKeyPair`, `derivePrivateKey`, checksum variant.
- Identities: `identityFromPublicKey`, `verifyIdentity`, `publicKeyFromIdentity`.
- Transactions: `serializeTransaction`, `signTransaction`, `verifyTransactionSignature`.
