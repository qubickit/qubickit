Add a row for each project you want to feature. Link to public repos, docs, or live sites whenever possible.

## Submission checklist

1. **Fork or branch** this repo.
2. **Edit** `apps/docs/content/docs/showcase.mdx` and append your entry to the table above (keep it alphabetical if multiple projects exist).
3. Include:
   - Project name linked to its homepage/repo.
   - The QubicKit packages you use.
   - A short description (2 sentences max) that calls out the unique integration.
4. If you have a blog post or case study, add a list item under “Deep dives” (optional).
5. Run `bun run --cwd apps/docs typecheck` to ensure the docs build still passes.
6. Open a PR with a short summary and optional screenshots.
